# SureDriver — External Setup Guide

Follow these steps in order. Each section ends with a **verify** step so you know it worked.

**Estimated time:** ~2 hours for full production path; ~15 minutes for phone testing only.

## 1. Phone testing (do this first)

Your phone cannot reach `localhost`. The API and phone must be on the **same Wi‑Fi**.

### Steps

1. Start the API on your Mac:
   ```bash
   cd ~/Projects/suredriver   # or ~/Documents/suredriver
   pnpm dev:api
   ```

2. Find your Mac's IP (System Settings → Network → Wi‑Fi → Details → IP Address), or run:
   ```bash
   ipconfig getifaddr en0
   ```

3. Edit `apps/mobile/.env`:
   ```env
   EXPO_PUBLIC_API_URL=http://192.168.0.112:3000
   ```
   Replace `192.168.0.112` with your actual IP.

4. Restart Expo with a clean cache:
   ```bash
   pnpm dev:mobile -- --clear
   ```

5. Allow port 3000 through macOS Firewall if prompted.

### Verify

- On your phone's browser, open `http://YOUR_IP:3000/health`
- You should see: `{"status":"ok","service":"suredriver-api"}`
- Log in to the app with owner `2348011111111` / PIN `1234`

## 2. GitHub repository

Render deploys from Git. Cursor mobile also needs a remote repo.

### Steps

1. Create a repo at [github.com/new](https://github.com/new) (e.g. `suredriver`, private recommended).

2. From your project folder:
   ```bash
   git add .
   git commit -m "Initial SureDriver MVP"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/suredriver.git
   git push -u origin main
   ```

### Verify

- `git remote -v` shows your GitHub URL
- Code is visible on github.com

## 3. Supabase Postgres

Free tier: 500 MB. Used **only as a database** — not Supabase Auth or Storage.

### Steps

1. Sign up at [supabase.com](https://supabase.com) → **New project**
   - Name: `suredriver`
   - Region: closest to Nigeria (e.g. `eu-west-1` or `eu-central-1`)
   - Set a strong database password (save it)

2. Go to **Project Settings → Database → Connection string → URI**
   - Choose **Session mode** (port 5432) for Prisma migrations
   - Copy the string; it looks like:
     ```
     postgresql://postgres.[ref]:[PASSWORD]@aws-0-eu-west-1.pooler.supabase.com:5432/postgres
     ```

3. For **Render production**, use the **Transaction pooler** URL (port 6543) with `?pgbouncer=true` appended — better for serverless cold starts.

4. **Local Postgres migration** (one-time, before first deploy):
   - In `apps/api/prisma/schema.prisma`, change:
     ```prisma
     provider = "postgresql"
     ```
   - Set `DATABASE_URL` in `apps/api/.env` to your Supabase URI
   - Run:
     ```bash
     pnpm db:migrate
     pnpm db:seed
     ```
   - Switch `schema.prisma` back to `sqlite` for local dev if you prefer, **or** keep Postgres for everything.

> **Note:** The existing SQLite migration won't apply to Postgres. You'll need a fresh `prisma migrate dev` against Supabase to generate Postgres-compatible SQL.

### Verify

- Supabase **Table Editor** shows tables: `User`, `Booking`, `Trip`, `Payment`, etc.
- Seed users appear in the `User` table

## 4. Render API hosting

Free tier spins down after 15 min idle (~1 min cold start on first request).

### Steps

1. Sign up at [render.com](https://render.com) → connect your GitHub account.

2. **New → Blueprint** → select your `suredriver` repo
   - Render reads `render.yaml` at the repo root
   - Or: **New Web Service** → connect repo → set manually (see below)

3. Set environment variables in the Render dashboard:

   | Variable | Value |
   |----------|-------|
   | `DATABASE_URL` | Supabase connection string (pooler URL for prod) |
   | `JWT_SECRET` | Auto-generated by Render, or any long random string |
   | `NOMBA_MOCK` | `true` initially, then `false` when Nomba is ready |
   | `NOMBA_BASE_URL` | `https://sandbox.nomba.com` (sandbox) |
   | `NOMBA_CLIENT_ID` | From Nomba dashboard (TEST for sandbox) |
   | `NOMBA_CLIENT_SECRET` | Private key from Nomba email |
   | `NOMBA_PARENT_ACCOUNT_ID` | Main (parent) account ID — used in `accountId` header |
   | `NOMBA_SUB_ACCOUNT_ID` | Hackathon sub-account — credits checkout to your team wallet |
   | `NOMBA_WEBHOOK_SECRET` | Signature key from Nomba webhook settings |
   | `PLATFORM_FEE_PERCENT` | `0.15` |

4. **Build command** (if not using blueprint):
   ```bash
   pnpm install && pnpm --filter @suredriver/shared-types build && pnpm --filter @suredriver/api build && cd apps/api && npx prisma migrate deploy
   ```

5. **Start command:**
   ```bash
   cd apps/api && node dist/src/main.js
   ```

6. Note your service URL, e.g. `https://suredriver-api.onrender.com`

### Keep-alive (optional)

Free Render sleeps after 15 min. Use [cron-job.org](https://cron-job.org) (free) to ping every 10 minutes:
```
GET https://suredriver-api.onrender.com/health
```

Supabase free tier pauses after 7 days idle — the same cron ping keeps it awake.

### Verify

- `curl https://suredriver-api.onrender.com/health` returns `{"status":"ok",...}`
- Render logs show no Prisma connection errors

### Point mobile at production API

In `apps/mobile/.env` (for TestFlight/builds, not Expo Go dev):
```env
EXPO_PUBLIC_API_URL=https://suredriver-api.onrender.com
```

## 5. Nomba payments

Start with **sandbox** before going live.

### Steps

1. Sign up at [developer.nomba.com](https://developer.nomba.com)
   - Sandbox works without a full merchant account for testing

2. In the Nomba dashboard (or hackathon email), get:
   - `clientId` → `NOMBA_CLIENT_ID`
   - Private key → `NOMBA_CLIENT_SECRET`
   - Parent `accountId` → `NOMBA_PARENT_ACCOUNT_ID`
   - Sub-account `accountId` → `NOMBA_SUB_ACCOUNT_ID`

   Hackathon 2026: submit webhook URL + sub-account ID at
   https://forms.gle/hKfBRHZiTGvU7LC59

3. Set in `apps/api/.env` (local) or Render (production):
   ```env
   NOMBA_MOCK=false
   NOMBA_BASE_URL=https://sandbox.nomba.com
   NOMBA_CLIENT_ID=your_test_client_id
   NOMBA_CLIENT_SECRET=your_test_private_key
   NOMBA_PARENT_ACCOUNT_ID=your_parent_account_id
   NOMBA_SUB_ACCOUNT_ID=your_sub_account_id
   NOMBA_WEBHOOK_SECRET=your_webhook_secret
   ```

4. Register a **webhook** in Nomba dashboard:
   - URL: `https://suredriver-api.onrender.com/payments/webhooks/nomba`
   - Event: `payment_success`
   - Copy the webhook secret → `NOMBA_WEBHOOK_SECRET`

5. For local webhook testing, use [ngrok](https://ngrok.com):
   ```bash
   ngrok http 3000
   # Register https://xxxx.ngrok.io/payments/webhooks/nomba in Nomba sandbox
   ```

### Verify (sandbox)

- Owner books → Pay opens Nomba checkout WebView (not auto-mock)
- After payment, booking status becomes `paid`
- Driver sees the job offer

### Going live

- Switch `NOMBA_BASE_URL` to `https://api.nomba.com`
- Complete Nomba merchant KYC
- Use production API keys

## 6. Ops contacts (WhatsApp & support)

Update placeholder numbers before pilot launch.

Edit `packages/shared-types/src/index.ts`:

```typescript
export const SUPPORT_PHONE = '+234XXXXXXXXXX';   // Your ops phone
export const SUPPORT_WHATSAPP = '234XXXXXXXXXX'; // No + prefix for wa.me links
export const KYC_WHATSAPP = '234XXXXXXXXXX';     // Where drivers send license/ID
```

Rebuild shared-types after changing:
```bash
pnpm --filter @suredriver/shared-types build
```

Restart mobile app to pick up changes.

### Driver onboarding flow (manual)

1. Driver signs up in app → adds bank details
2. Driver sends license + national ID to your **KYC WhatsApp**
3. You approve via admin API:
   ```bash
   # Login as admin, get token, then:
   curl -X PATCH https://YOUR_API/admin/drivers/DRIVER_USER_ID/verification \
     -H "Authorization: Bearer ADMIN_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"status":"approved"}'
   ```

### Invite new users

```bash
curl -X POST https://YOUR_API/admin/invites \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"phone":"2348033333333","role":"owner"}'
```

## 7. Expo account (optional)

Needed for: standalone iOS/Android builds, reliable push notifications, App Store later.

### Steps

1. Sign up at [expo.dev](https://expo.dev)

2. Create a project:
   ```bash
   cd apps/mobile
   npx eas init
   ```

3. Add to `apps/mobile/app.json`:
   ```json
   "extra": {
     "eas": {
       "projectId": "your-project-id-from-eas"
     }
   }
   ```

4. Add `expo-notifications` plugin to `app.json` plugins array (for push in standalone builds).

### For now (Expo Go)

Expo Go on SDK 54 works without an Expo account. Push notifications are best-effort in Expo Go; the app polls every 5 seconds as a fallback.
